# Sprint Review 演示脚本

## 1. 演示目标

在 5 到 8 分钟内演示 Sprint 1 已经打通的闭环：

`Prompt -> 页面生成 -> 校验展示 -> AI 二次修改 -> 固化保存`

## 2. 演示前准备

- 启动 MCP UI Server
- 启动 Flutter Studio
- 确认服务端已有写权限，SQLite 数据库可保存版本
- 打开 Studio 首页，确保左上状态正常

## 3. 推荐演示路径

### Step 1：说明当前目标

讲解一句话：

`本期目标是让用户输入一句需求，系统生成单页 DSL，并在 Studio 中继续编辑和固化。`

### Step 2：输入 Prompt 生成页面

输入：

`帮我生成一个客户列表页，包含搜索、状态筛选、负责人列和操作按钮`

预期展示：

- 右侧生成摘要出现
- 中间预览区出现页面
- JSON 编辑区同步出现 DSL

### Step 3：查看校验结果

强调两点：

- Studio 会自动触发页面校验
- 若有错误或警告，会明确显示路径和建议

### Step 4：执行 AI 二次修改

输入：

`把标题改成客户运营列表，并增加搜索筛选和操作按钮`

预期展示：

- 标题更新
- 搜索/筛选条出现
- 变更摘要列出已应用修改

### Step 5：人工微调

演示任选一种：

- 在 JSON 编辑器中直接修改一段文字
- 添加一个区块
- 调整区块顺序

### Step 6：固化保存

点击：

- `固化当前页`

预期展示：

- 版本号生成
- 左侧版本列表刷新
- 页面可重新加载

## 4. 演示要点

- 不强调“AI 完全自动”，而强调“AI 初稿 + Studio 校对”
- 强调生成结果是结构化 DSL，不是纯文案
- 强调服务端校验和版本化能力
- 强调当前范围仅限单页 MVP

## 5. 失败兜底

如果生成结果不理想，使用下面方案：

- 切换到已生成过的稳定页面继续演示二次修改
- 直接展示样例页的 AI 二次修改与固化
- 用 JSON 编辑区演示人工可修正性

## 6. 演示结束总结

结尾建议：

`Sprint 1 已完成单页生成、校验、预览、AI 二次修改和固化保存闭环，下一阶段将进入更完整的 Studio 冒烟验证与多页应用建模。`
